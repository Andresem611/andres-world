<task type="auto">
  <name>Task 1: Add vercel.json and push to trigger CI/CD deploy</name>
  <files>vercel.json</files>
  <action>
Create `vercel.json` at the project root with the SPA rewrite rule shown in the interfaces block. This is the only file change needed for Vercel configuration.

Without this file, Vercel will attempt to resolve URL paths to files in `dist/`. Since this is a single-page app, any path other than `/` will 404 on page refresh. The rewrite routes all requests to `index.html` and lets the client-side app handle routing.

After creating the file, commit and push to `main`:

```bash
git add vercel.json
git commit -m "feat(infra): add vercel.json SPA rewrite"
git push origin main
```

This push triggers Vercel CI/CD automatically (assumes Vercel project is already connected to this GitHub repo — if not, the checkpoint task below handles it).

Vercel auto-detects Vite. Build settings should be:
- Framework Preset: Vite (or "Other" with Output Directory: `dist`)
- Build Command: `npm run build`
- Output Directory: `dist`

These match the `package.json` scripts created in Plan 01.
  </action>

  </verify>
